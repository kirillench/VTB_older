# MultiBank - Мультибанковский финансовый агрегатор

## Описание проекта

MultiBank - это платформа для агрегации финансовых данных из различных банков через Open Banking API. Проект предоставляет пользователям единую картину их финансов, аналитику расходов, прогнозирование и управление бюджетом.

## Ценность для пользователя

### Основные функции:

1. **Агрегация счетов и карт**
   - Объединение всех счетов из разных банков в едином интерфейсе
   - Отображение общего баланса и балансов по каждому счету
   - Поддержка нескольких валют

2. **Финансовая аналитика**
   - Анализ расходов по категориям
   - Тренды расходов за месяц
   - Топ категорий расходов
   - Визуализация данных через графики и диаграммы

3. **Управление бюджетом**
   - Создание бюджетов по категориям
   - Отслеживание использования бюджета
   - Уведомления о превышении лимитов

4. **Прогнозирование**
   - Прогноз баланса на основе исторических данных
   - Анализ трендов доходов и расходов

## Техническая архитектура

### Backend (Go + Gin + GORM + PostgreSQL):
- RESTful API
- OAuth 2.0 для подключения банков
- Шифрование токенов (AES-GCM)
- Автомиграция базы данных
- Обработка ошибок API банков

### Frontend (React + Vite + TailwindCSS + Recharts):
- Адаптивный дизайн
- Интерактивные графики и визуализации
- Компонентная архитектура
- Контекст для управления состоянием

### База данных:
- PostgreSQL для хранения данных
- Модели: User, UserBank, Account, Transaction, Subscription

## Масштабируемость

### Архитектурная гибкость:

1. **Поддержка новых банков**
   - Модульная архитектура OAuth клиентов
   - Легкое добавление новых провайдеров через конфигурацию
   - Единый интерфейс для работы с разными API

2. **Горизонтальное масштабирование**
   - Stateless API серверы
   - Возможность добавления балансировщиков нагрузки
   - Кэширование данных

3. **Расширяемость функционала**
   - Модульная структура handlers
   - Легкое добавление новых endpoints
   - Плагинная архитектура для аналитики

## Безопасность

1. **Аутентификация**
   - Хеширование паролей (bcrypt)
   - Сессионные токены (в демо-режиме)
   - Готовность к переходу на JWT

2. **Шифрование данных**
   - AES-GCM шифрование токенов доступа
   - Хранение зашифрованных токенов в БД
   - Безопасная передача данных

3. **Обработка ошибок**
   - Корректная обработка ошибок API банков
   - Логирование ошибок
   - Graceful degradation при недоступности сервисов

## Использование Open Banking API

- Интеграция с sandbox API: https://vbank.open.bankingapi.ru
- OAuth 2.0 flow для получения consent
- Получение данных о счетах и транзакциях
- Обработка ошибок и лимитов API
